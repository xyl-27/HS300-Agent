# 数据获取模块
stock_info 是固定从csv导入的,
stock_daily 是检查数据库是否有的,无则从20250101开始导入,有则根据最新的增量导入
## 1. 数据获取流程

### 1.1 数据来源优先级
1. 首先查询数据库，如有数据直接返回
2. 数据库无数据时，从 `hs300_data.csv` 文件读取

### 1.2 使用接口
akshare库
- `stock_zh_a_hist`：获取股票历史交易数据
- `stock_individual_info_em`：获取股票基本信息

## 2. 实现方案

### 2.1 核心组件
- 创建 `DataCollector` 实例用于统一管理股票数据获取

### 2.2 数据时间范围
- 默认时间范围：2025年至今（日频数据）

### 2.3 成分股列表处理
- 从 `hs300_data.csv` 文件读取沪深300成分股信息
- CSV文件包含字段：股票代码、股票名称、上市时间等

### 2.4 单只股票处理流程
1. 遍历每只成分股（共300只）
2. 显示当前处理进度（格式：处理第X/300只股票: 股票代码 - 股票名称）
3. 读取股票的上市时间
4. 解析上市年份信息

### 2.5 数据获取策略
- 按年份分批获取数据
- 对每个年份（2025-当前年）分别请求数据
- 跳过股票上市前的年份
- 构建年度起止日期：如 20250101 到 20251231

## 3. 技术实现要点
- 使用 akshare 库进行数据接口调用
- 实现数据缓存机制，避免重复请求
- 处理网络异常和数据缺失情况
- 优化数据获取效率，减少请求次数
